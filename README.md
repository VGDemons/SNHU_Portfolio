Airgead Banking Investment Calculator - Portfolio Reflection
Project Summary
This project is a console-based investment calculator application for Airgead Banking that helps users visualize the growth of their investments over time. The application calculates compound interest with and without additional monthly deposits, displaying year-end balances and earned interest in a formatted report. The problem it solves is providing customers with a clear financial projection tool to understand how different variables (initial investment, monthly deposits, interest rates, and time horizons) impact their investment growth.

What I Did Particularly Well
I implemented robust input validation using custom parsing functions that safely convert user input to numeric values while preventing crashes from invalid entries. The code demonstrates excellent separation of concerns with distinct classes for user interface (AirgeadBankingApp) and financial calculations (InvestmentCalculator). The application follows consistent coding standards throughout, including proper use of header guards, naming conventions (m_ prefix for members, t_ prefix for parameters), and C++ style casts instead of C-style casts. The error handling is comprehensive, using try/catch blocks as specified in the requirements.

Where I Could Enhance My Code
The code could be enhanced by implementing a more modular design for the report generation, possibly extracting the display formatting into a separate ReportFormatter class. This would make the code more maintainable if additional report types are needed in the future. Efficiency could be improved by implementing input validation with regular expressions instead of multiple parsing attempts. For security, I could add input sanitization to prevent potential buffer overflow issues, though the current implementation already uses safe string handling. The calculations could be enhanced with support for different compounding frequencies (daily, quarterly) rather than just monthly.

Most Challenging Code and How I Overcame It
The most challenging part was implementing the compound interest calculations with proper handling of the monthly deposits and ensuring the year-end reporting was accurate. I overcame this by breaking down the problem into smaller components - first implementing the basic calculation without deposits, then adding the deposit functionality, and finally ensuring the reporting aggregated correctly. I added detailed comments to document the formula implementation and created a separate calculation method that could be called with different deposit parameters. The tools I added to my support network include using std::fixed and std::setprecision for consistent financial formatting, and implementing safe parsing functions that handle edge cases like overflow and invalid input.

Transferable Skills
The skills from this project that are particularly transferable include input validation techniques, separation of concerns in object-oriented design, and implementing financial calculations with proper precision. The experience with console application development and user interface design will be valuable for future projects requiring command-line interfaces. The error handling patterns and defensive programming techniques are applicable across all software development contexts.

Maintainability, Readability, and Adaptability
I made this program maintainable by following consistent naming conventions, adding comprehensive comments, and separating business logic from presentation. The code is readable through clear method names, logical organization of classes, and consistent formatting. The design is adaptable through the use of a separate calculation class that could be extended with additional financial products, and the input/output class that could be modified to support different interfaces (like GUI or web) without changing the core calculation logic. The use of standard C++ containers and algorithms ensures the code can be adapted to different platforms and compilers.
